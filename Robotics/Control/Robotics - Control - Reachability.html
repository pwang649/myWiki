Reachability and Safety

The Reachability Problem and Connections with Safety​

We are now ready to look into the key part, i.e. how can we compute unsafe sets for an autonomous system and how can we design a safety-preserving controller?

There are many ways to compute these unsafe sets but we will primarily look into the Hamilton-Jacobi Reachability method to compute unsafe sets and safety-preserving controllers. The reason to study HJ reachability is its ability to seamlessly consider control bounds, uncertainty, or even state constraints while computing unsafe sets. Moreover, the reachability method is applicable to general nonlinear systems.

Let's start by looking into what the reachability problem is and then we will see how it is connected with the safety analysis of autonomous systems.

Reachability Problem​

In reachability, we are interested in computing the Backward Reachable Tube (BRT) of a dynamic system. BRT is the set of all starting states of the system that will eventually reach some target set despite the best control effort.

If the target set represents the failure set of a system then BRT represents the potential unsafe set of states for the system and thus should be avoided. Pictorially:

Conversely, the complement of BRT represents the set of safe states for the system.

Mathematically, let

L⊆Rn\mathcal{L} \subseteq \mathbb{R}^{n}L⊆Rn be the target set (typically failure set)

BRT(t)⊆RnB R T(t) \subseteq \mathbb{R}^{n}BRT(t)⊆Rn be the BRT at time ttt (typically unsafe set)

BRT(t)={x:∀u(⋅)∈UtT,∃d(⋅)∈DtT,ξx,tu,d(s)∈LB R T(t)=\left\{x: \forall u(\cdot) \in \mathbb{U}_{t}^{T}, \exists d(\cdot) \in \mathbb{D}_{t}^{T}, \xi_{x, t}^{u, d}(s) \in \mathcal{L}\right.BRT(t)={x:∀u(⋅)∈UtT​,∃d(⋅)∈DtT​,ξx,tu,d​(s)∈L for some s \in[t, T]\right\}

Intuitively, BRT(t)B R T(t)BRT(t) computes the set of all starting states from which no matter what control does, there exists a disturbance, that will drive the system inside the target set (or the failure set).

In other words, to compute the unsafe set we need to compute the BRT of the failure set.

Examples of the Failure Sets​

  • A high turbulence region for an aircraft.

  • Ceiling, floor for an indoor autonomous drone.

  • A no-go zone for a warehouse robot.

  • An unmapped area for an autonomous car.

Hamilton-Jacobi Reachability​

Now that we have established a connection between the BRT and the unsafe set of a system, let's talk about how we can compute the BRT. Once again, there are many methods proposed to compute the BRT; we will particularly look into HJ reachability that formulates the BRT computation as an optimal control problem. This allows us to use all the optimal control tools that we have learned so far to compute the BRT.

Specifically, HJ reachability leverages level set methods to convert the BRT computation into an optimal control problem. First, we define a target function l(x)l(x)l(x) to implicitly represent the target set L\mathcal{L}L, i.e., L={x:l(x)≤0}\mathcal{L}=\{x: l(x) \leq 0\}L={x:l(x)≤0}. In other words,

l(x)≤0⇔x∈Ll(x) \leq 0 \Leftrightarrow x \in \mathcal{L}l(x)≤0⇔x∈L

Thus, the target set is given by the subzero level of the target function l(x)l(x)l(x).

  • One such function is signed distance to L\mathcal{L}L.
    • Signed-distance >0> 0>0 when xxx is outside L\mathcal{L}L
    • Signed-distance <0< 0<0 when xxx is inside L\mathcal{L}L
    • Signed-distance =0= 0=0 when xxx is at the boundary of L\mathcal{L}L
  • As an example, imagine the target set is a circle of radius RRR centered at the origin.

Converting to an OC Problem​

Recall that we are formulating BRT computation as an optimal control problem. For an optimal control problem, we need a cost function. To do so, we define the following cost function:

J(x,u(⋅),d(⋅),t)=min⁡s∈[tsT]l(x(s))J(x, u(\cdot), d(\cdot), t)=\min _{s \in\left[t_{s} T\right]} l(x(s))J(x,u(⋅),d(⋅),t)=s∈[ts​T]min​l(x(s))

JJJ represents the minimum signed distance achieved by the trajectory starting at xxx during the time horizon [t,T][t, T][t,T] under the control u(⋅)u(\cdot)u(⋅) and disturbance d(⋅)d(\cdot)d(⋅). Let's understand this cost function a bit better.

  • Case 2:

    J(x0,u2(⋅),d2(⋅),t)<0J\left(x_{0}, u_{2}(\cdot), d_{2}(\cdot), t\right)<0J(x0​,u2​(⋅),d2​(⋅),t)<0

  • Case 3:

    J(x0,u3(⋅),d3(⋅),t)<0J\left(x_{0}, u_{3}(\cdot), d_{3}(\cdot), t\right)<0J(x0​,u3​(⋅),d3​(⋅),t)<0

Thus, by looking at the sign of JJJ, we can tell whether the trajectory ever entered the target set or not under a given control and disturbance function. If we want to keep the system safe, the control should try to keep the system outside the target set as much as it can, i.e., it should try to maximize JJJ and the disturbance should try to minimize JJJ. Thus, our optimal control problem reads:

Now suppose that V(x0,t)<0V\left(x_{0}, t\right)<0V(x0​,t)<0 for some state x0x_{0}x0​. This means that the control tried but couldn't do anything to avoid getting the system into the target set. In other words, x0∈BRT(t)x_{0} \in B R T(t)x0​∈BRT(t).

Similarly, if V(x0,t)>0V\left(x_{0}, t\right)>0V(x0​,t)>0. This means that the control was successful in keeping the system trajectory outside the target set despite the worst-case uncertainty. In other words, x0∉BRT(t)x_{0} \notin B R T(t)x0​∈/BRT(t).

Thus, once we have the value function, we can compute the BRT as:

BRT(t)={x:V(x,t)≤0}B R T(t)=\{x: V(x, t) \leq 0\}BRT(t)={x:V(x,t)≤0}

Solving the OC Problem​

Great! We now have an optimal control problem whose solution can give us the BRT. So all that remains is to solve the optimal control problem. The issue is that the cost function of this OC problem is different -- it is no longer a running cost as we had earlier, rather it is a minimum operation over time. The good news is that we can still use the principle of optimality and dynamic programming to compute the value function.

Recall that,
