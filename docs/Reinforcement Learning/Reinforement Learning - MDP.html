Markov Decision Process

Markov Property​

The future is independent of the past given the present

  • State Transition Matrix
    • For a markov state s and successor state s′s^{'}s′, the state transition probability is defined by
      Pss′=P[St+1=s′∣St=s]\begin{gather*} P_{ss^{'}} = \mathrm{P}[S_{t+1} = s^{'}|S_t = s] \end{gather*}Pss′​=P[St+1​=s′∣St​=s]​
    • The complete state transition matrix PPP defined transition probabilities from all states s to all successor states s′s^{'}s′

Markov Process​

A markov process is a memoryless random process, i.e. a sequence of random states S1,S2,...S_1, S_2, ...S1​,S2​,... with markov property

  • Definition

    • A markov process or markov chain is a tuple (S,P)(S,P)(S,P)
      • S is a (finite) set of states an agent can be in
      • PPP is a state transition probability matrix
      • Pss′=P[St+1=s′∣St=s]P_{ss^{'}} = \mathrm{P}[S_{t+1} = s^{'}|S_t = s]Pss′​=P[St+1​=s′∣St​=s]
      • The dynamics of an environment can be explained by markov process

  • A sample is a sequence of states of a markov chain

Markov Reward Process​

It is a markov chain with values i.e. value judgements

  • Definition
    • A markog reward process is a tuple <S,P,R,γ><S,P,R,\gamma><S,P,R,γ>
      • S is a (finite) set of states an agent can be in
      • PPP is a state transition probability matrix
      • Pss′=P[St+1=s′∣St=s]P_{ss^{'}} = \mathrm{P}[S_{t+1} = s^{'}|S_t = s]Pss′​=P[St+1​=s′∣St​=s]

      • γ\gammaγ is a discount factor, γ∈[0,1]\gamma \in [0,1]γ∈[0,1]

Return (Goal)​

  • Definition
    • The return GtG_tGt​ is the total discounted reward from time-step ttt
    • Gt=Rt+1+γRt+2=∑k=0inf⁡γkRt+k+1G_t = R_{t+1} + \gamma R_{t+2} = \sum_{k=0}^{\inf} \gamma^{k} R_{t+k+1}Gt​=Rt+1​+γRt+2​=∑k=0inf​γkRt+k+1​
    • There is no expectation here because we are talking about a random sample i.e. GtG_tGt​ is just one sample from our markov reward process
    • The discount factor γ∈[0,1]\gamma \in [0,1]γ∈[0,1] is the present value of the future rewards
    • γ=0\gamma = 0γ=0 is short sighted and γ=1\gamma = 1γ=1 is far sighted
    • The value of receiving reward RRR after k+1k+1k+1 time-steps is γkR\gamma^{k} RγkR
Why discount factor ?​
  • Mathematically convenient to discount rewards
  • Avoids infinite returns in cyclic markov processes
  • Uncertainty about the future may not be fully represented
  • If the reward is financial, immediate rewards may earn more interest than delayed rewards
  • Animal/human behaviour shows preference for immediate reward
  • It is sometimes possible to use undiscounted markov reward processes i.e. γ=1\gamma = 1γ=1, e.g. if all sequences terminate. Therefore by definition all returns are finite

Bellman Equation for MRPs​

This is the most fundamental relation in RL. The main idea is that the value function can be decomposed into two parts, which can be used recursive:

  • immediate reward Rt+1R_{t+1}Rt+1​

  • dicount value of successor state γv(St+1)\gamma v(S_{t+1})γv(St+1​)

  • math breakdown

    V(s)=E[Gt∣St=s]=E[Rt+1+γRt+2+γ2Rt+3∣St=s]=E[Rt+1+γ(Rt+2+γRt+3)∣St=s]=E[Rt+1+γGt+1∣St=s]=E[Rt+1+γv(St+1)∣St=s]\begin{align*} V(s) & = E[G_t | S_t =s] \\ & = E[R_{t+1} + \gamma R_{t+2} + \gamma^{2} R_{t+3}| S_t =s] \\ & = E[R_{t+1} + \gamma (R_{t+2} + \gamma R_{t+3})| S_t =s] \\ & = E[R_{t+1} + \gamma G_{t+1}| S_t =s] \\ & = E[R_{t+1} + \gamma v(S_{t+1})| S_t =s] \end{align*}V(s)​=E[Gt​∣St​=s]=E[Rt+1​+γRt+2​+γ2Rt+3​∣St​=s]=E[Rt+1​+γ(Rt+2​+γRt+3​)∣St​=s]=E[Rt+1​+γGt+1​∣St​=s]=E[Rt+1​+γv(St+1​)∣St​=s]​
    • The bellman equation is v(s)=E[Rt+1+γv(St+1)∣St=s]v(s)= E[R_{t+1} + \gamma v(S_{t+1})| S_t =s]v(s)=E[Rt+1​+γv(St+1​)∣St​=s]
    • So by definition if we got a value function correct it must obey bellman equation
Bellman Equation - Matrix Form​
  • Concise matrix notation is v=R+γPvv = R + \gamma P vv=R+γPv
    • where vvv is a column vector with one entry per state

Solving the Bellman Equation​

  • The bellman equation is a linear equation
  • It can be solved directly:
    • v=R+γPv→v=(I−γP)−1Rv = R + \gamma Pv \rightarrow v = (I - \gamma P)^{-1} Rv=R+γPv→v=(I−γP)−1R
  • It is a process of evaluating the rewards
  • Computational complexity is O(n3)O(n^3)O(n3) for nnn states
  • Direct solution only possible for small MRPs
  • There are many iterative methods for large MRPs, e.g.
    • Dynamic Programming
    • Monte-Carlo evaluation
    • Temporal-Difference learning

What is the intuition behind the Bellman Optimality Equation​

The q∗q_{*}q∗​ or v∗v_{*}v∗​ is telling you what is the maximum score you can get in a given state (say a moment of the atari game). The intution is that, all you need to do to get that maximum score is the principle of optimality, which tell you that the way to get optimal score is to behave optimally for one step and to behave optimally for the reminder of your trajectory. If you behave optimally for the reminder of your trajectory that is the value function from the state you end up in. Now, all you need to do is to figure out how to behave optimally for one step, and the way to do this is to maximize over those optimal value functions of the places you might end up in.

Basically, breaking down your trajectory into two parts

  • Optimal decision at one step
  • Optimal decision from there onwards
