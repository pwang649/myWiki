PAC Learning

Definition (PAC-learnability)​

A hypothesis class HHH is PAC-learnable if there exists a learning algorithm with the following property: For every ϵ,δ∈(0,1)\epsilon, \delta \in (0, 1)ϵ,δ∈(0,1), every distribution DDD over XXX, and every h∈Hh \in Hh∈H, when the algorithm is given m(ϵ,δ)m(\epsilon, \delta)m(ϵ,δ) samples drawn from DDD and labeled by hhh, then the algorithm produces a hypothesis h^\hat{h}h^ such that with probability 1−δ,L(h^)≤ϵ1 - \delta, L(\hat{h}) \leq \epsilon1−δ,L(h^)≤ϵ. (Note that the probability is over randomness in the training set as well as any internal algorithmic randomness).

Agnostic PAC-Learning​

  • PAC learning requires the realizibility assumption, meaning that it cannot handle noise / error within the labels.
  • Agnostic PAC learning relaxes the realizability distribution, and also allows the labels to be noisy.
  • PAC learnable implies agnostic PAC learnable. Therefore Agnostic PAC learning is more general than PAC learning.

Definition (Uniform Convergence)​

We say that a hypothesis class HHH has the uniform convergence property (w.r.t. a domain ZZZ and a loss function lll) if there exists a function mHUC:(0,1)2→Nm_H^{UC}:(0, 1)^2 \to \mathbb{N}mHUC​:(0,1)2→N such that for every ϵ,δ∈(0,1)\epsilon, \delta \in (0, 1)ϵ,δ∈(0,1) and for every probability distribution DDD over ZZZ, if SSS is a sample of m≥mHUC(ϵ,δ)m \ge m_H^{UC}(\epsilon, \delta)m≥mHUC​(ϵ,δ) examples drawn i.i.d. according to DDD, then, with probability of at least 1−δ1-\delta1−δ, SSS is ϵ\epsilonϵ-representative.
